以下产品决策已拍板，不要重新讨论：
1. 搜索第一版 = 成员消息 + Agent 最终回复；不包含命令过程、状态事件、文件变化等过程事件。

项目背景也已确认：
- SyncAI 不是重做 AI 编程工具，而是复用 Codex / Claude Code 的底层能力，做面向小团队的 AI 编程协作与共享平台。
- MVP 结构为 团队 > 项目 > 会话。
- 项目层保留但做轻量。
- 项目下可有多个会话；会话分共享 / 私有；共享会话项目成员可见且都可继续发言；私有会话挂在项目下且仅创建者可见；新加入项目成员默认可看历史共享会话；共享 / 私有之间允许互转。
- Todo 只做轻量 todo，从消息手动提取，字段仅 标题 + 状态；状态仅 待办 / 已完成；保留来源消息跳转；展示在会话侧边栏。
- 团队角色只区分管理员 / 普通成员；团队成员默认都能进入所有项目；第一版支持项目归档。
