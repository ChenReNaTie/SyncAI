# SyncAI / 灵悉 AI - 需求澄清结果

**更新时间：** 2026-04-14
**状态：** 已根据用户多轮问答确认，优先级高于早期草案文档中的未确认内容

---

## 1. 产品最新定位

- SyncAI 不是一个新的 AI 编程工具。
- SyncAI 不重做 Claude Code 或 Codex。
- SyncAI 的定位是：**面向小团队的 AI 编程协作与共享平台**。
- 底层编程能力直接复用：
  - Claude Code
  - Codex
- SyncAI 负责的上层能力：
  - 共享 AI 会话历史
  - 同步中间过程和最终结果
  - 让团队成员在同一个会话里继续发消息
  - 从对话中提取轻量 todo
  - 搜索和复用历史上下文

---

## 2. 目标用户与核心痛点

- 目标用户：**小团队协作**
- 第一核心痛点：**AI 对话历史不能共享**

---

## 3. MVP 结构与层级

MVP 采用以下结构：

- 团队
  - 项目
    - 会话

说明：

- 项目层保留，但做轻量，不做复杂项目管理功能。
- 一个团队内可以有多个项目。
- 一个项目下可以有多个会话。

---

## 4. 会话模型

### 4.1 会话类型

- 共享会话
- 私有会话

### 4.2 共享会话规则

- 项目内所有成员都能看到共享会话历史。
- 项目内所有成员都可以继续发言和追问 AI。
- 新加入项目的成员，默认可以看到该项目下过去的共享会话历史。

### 4.3 私有会话规则

- 私有会话挂在项目下面。
- 私有会话只有创建者自己可见。

### 4.4 会话转换

- 私有会话可以一键转共享。
- 共享会话也可以转回私有。
- 但产品规则需单独说明：如果共享期间已有成员看过历史，则“转回私有”仅影响后续可见性，不代表历史从未暴露。

### 4.5 会话列表页第一版显示字段

- 会话标题
- 类型（共享 / 私有）
- 最后更新时间

### 4.6 会话标题策略

- 第一版由用户手动填写。

### 4.7 会话增强能力

- 第一版支持置顶
- 第一版支持收藏

---

## 5. 团队与成员规则

### 5.1 团队创建与加入

- 用户先创建团队，再手动邀请成员。

### 5.2 团队角色

第一版区分两种角色：

- 管理员
- 普通成员

### 5.3 项目成员规则

- 团队成员默认都能进入所有项目。
- 不单独做复杂的项目成员权限隔离。

### 5.4 项目归档

- 第一版支持项目归档。

---

## 6. Todo 设计（MVP）

### 6.1 Todo 生成方式

- 手动从某条消息中“提取为待办”。
- 不做 AI 自动建议待办。

### 6.2 Todo 最小字段

- 标题
- 状态

### 6.3 Todo 状态

第一版只保留两个状态：

- 待办
- 已完成

### 6.4 Todo 与消息关系

- 每个 todo 要保留来源消息跳转。
- 点击 todo 能跳回原消息。

### 6.5 Todo 展示位置

- 第一版放在当前会话的侧边栏 / 侧区。

---

## 7. AI / Agent 集成要求

### 7.1 底层 Agent 支持

- 第一版同时支持 Claude Code 和 Codex。
- 但单个会话一次只绑定一个底层 agent。

### 7.2 默认模型 / Agent 入口

- 第一版只有一个默认模型入口。
- 不做多模型切换 UI。

### 7.3 团队成员发指令规则

- 项目内所有成员都可以往同一个共享会话里继续发消息。

### 7.4 消息进入底层 Agent 的方式

- 第一版采用直接追加到同一个会话继续运行的方式。
- 由于多人可能同时发消息，底层实现仍需有队列 / 串行调度能力。

### 7.5 要共享的内容层级

用户明确希望在平台上展示的内容结构，与 Claude Code / Codex 终端中的内容结构保持一致，但样式由 SyncAI 自己设计。

第一版至少要共享：

- 最终回复
- 关键步骤日志
- 命令执行过程
- 状态变化

说明：

- 用户要的是“内容结构一致”，不是“终端样式完全一致”。
- 也就是说，Claude Code / Codex 显示哪些内容层级，SyncAI 页面上也要显示对应层级。
- 但样式、颜色、布局由 SyncAI 自己设计。

### 7.6 运行状态显示

第一版要显示 agent 执行状态，至少包括：

- 正在思考
- 执行中
- 已完成
- 出错

### 7.7 消息编辑 / 删除

- 第一版不支持消息编辑。
- 第一版不支持消息删除。

---

## 8. 搜索能力

### 8.1 搜索范围

- 第一版需要做项目级 / 全局搜索。

### 8.2 搜索结果最少显示字段

- 命中的消息内容
- 所属会话
- 所属项目
- 命中人
- 时间

---

## 9. 已明确不做的内容（MVP）

第一版明确不做：

- IDE 插件
- 企业权限体系
- 复杂审批流
- 多模型编排

---

## 10. 统一事件模型（建议）

为兼容 Claude Code / Codex，平台内部建议统一抽象以下事件类型：

- Session 事件
- Message 事件
- Status 事件
- Tool / Command 事件
- File Change 事件
- Todo 事件

该模型用于：

- 统一前端展示
- 搜索和存档
- Todo 提取
- 协作回放
- 多 Agent 兼容

---

## 11. 当前结论的产品定义

SyncAI 第一版 = **面向小团队的 AI 编程协作与共享平台**。

它的核心不是重做 AI 编程器，而是复用 Claude Code / Codex，把原本只存在于个人终端里的 AI 编程过程，转化为团队可共享、可追踪、可搜索、可协作的项目资产。
