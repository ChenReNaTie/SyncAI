# SyncAI（灵悉 AI）启动说明

## 前置条件

| 项目 | 要求 | 怎么看版本 |
|------|------|-----------|
| Node.js | >= 24.0.0 | `node -v` |
| Docker Desktop | 最新稳定版即可 | `docker --version`，任务栏有没有鲸鱼图标 |
| npm | 随 Node.js 自带 | `npm -v` |
| Git | 任意较新版本 | `git --version` |

> ⚠️ **Node.js 版本必须 >= 24。** 如果用更低版本，`package.json` 里的 `engines` 字段会直接报错。
>
> ⚠️ **真实 Codex 模式需要本机可用的 Codex CLI。** 项目默认直接使用当前电脑上已经安装好的 `codex`，也支持手动指定路径。
>
> ⚠️ **这不是可选项。** 当前 SyncAI 的真实会话执行依赖 Codex；换一台电脑继续跑时，也必须在新电脑上准备好 Codex 运行环境和认证状态。

---

## 第一步：安装依赖

在项目根目录执行：

```bash
npm install
```

这一步会安装所有 workspace 的依赖（`@syncai/server`、`@syncai/web`、`@syncai/shared`）。

如果你要跑真实 Codex，建议同时确认本机能直接执行：

```bash
codex --version
```

如果这条命令能成功，说明你的 Codex CLI 安装基本正常。

建议再执行一次环境自检：

```bash
npm run doctor
```

如果你想顺便验证当前电脑上的 Codex 是否已经具备真实执行能力，可以再运行：

```bash
npm run doctor -- --with-codex-exec
```

> 如果 `npm install` 失败，先确认 Node.js 版本 ≥ 24，然后删除 `node_modules` 重试。

---

## 第二步：启动数据库

SyncAI 用 PostgreSQL + Redis 做存储和会话管理，部署在 Docker 里：

```bash
npm run db:up
```

这个命令等价于 `docker compose up -d postgres redis`，会以 **detached 模式**启动两个容器：

| 容器 | 镜像 | 端口 | 默认密码 |
|------|------|------|---------|
| syncai-postgres | postgres:16-alpine | 5432 | syncai |
| syncai-redis | redis:7-alpine | 6380 | 无密码 |

### 数据库默认连接信息

- **PostgreSQL：** `postgres://syncai:syncai@127.0.0.1:5432/syncai`
- **Redis：** `127.0.0.1:6380`（无密码）

验证数据库已经起来了：

```bash
docker ps --filter "name=syncai"
```

你应该看到 `syncai-postgres` 和 `syncai-redis` 两个容器的状态是 `healthy`。

---

## 第三步：执行数据库迁移（建表）

```bash
npm run db:migrate
```

这一步会在 PostgreSQL 中创建所有需要的表。**每个 schema 变更后也需要重新跑一次这个命令。**

---

## 第四步：启动项目

```bash
npm run dev
```

这个命令会**同时启动前后端**（通过 `scripts/dev.mjs` 编排）：

| 服务 | 地址 | 说明 |
|------|------|------|
| 前端（Vite） | `http://localhost:5173` | React SPA，浏览器直接访问 |
| 后端（API） | `http://localhost:3001` | Express API 服务 |

前端 `/api` 开头的请求会自动代理到后端 `3001` 端口（Vite proxy 配置），所以前端代码里直接用 `/api/v1/xxx` 即可，不用写完整 URL。

---

## 验证启动成功

### 方法 1：前端页面

浏览器打开 `http://localhost:5173`，应该看到**登录 / 注册页面**。

如果没有自动跳转，直接访问 `http://localhost:5173/login`。

### 方法 2：API 健康检查

```bash
curl http://localhost:3001/api/v1/health
```

返回类似 `{"status":"ok"}` 就说明后端正常。

---

## 停止项目

- 停止前后端：按 `Ctrl + C` 退出 `npm run dev`
- 停止数据库：`npm run db:down`（对应 `docker compose down`）
  - 数据卷（volume）**不会删除**，下次 `db:up` 数据还在
  - 如果想彻底清空数据：`docker compose down -v`

## 常见问题

### 1. Docker 没启动 / Docker Desktop 没运行

**现象：** `npm run db:up` 报 `docker: command not found` 或 `Cannot connect to the Docker daemon`

**解决：** 启动 Docker Desktop（Windows 任务栏右下角的鲸鱼图标），等它变成稳定状态后再执行命令。Linux/macOS 可以用 `sudo systemctl start docker`。

### 2. 端口被占用

**现象：** `Port 5432 is already in use`、`Port 6379 is already in use`、`Port 3001 is already in use`、`Port 5173 is already in use`

**解决：**

- **5432/6379：** 说明本机已有 PostgreSQL 或 Redis 在跑。可以关掉本机的，或者修改 `docker-compose.yml` 里的端口映射（比如改成 `5433:5432`），然后修改后端的环境变量 `SYNCAI_POSTGRES_PORT=5433`。
- **3001：** 说明有别的程序占了 3001 端口。找到并关掉它，或者设环境变量 `PORT=3002` 让后端用别的端口。
- **5173：** Vite 默认端口。如果被占，Vite 会自动尝试 5174、5175……，注意看终端输出。

### 3. 数据库连接失败

**现象：** 后端启动后报 `connect ECONNREFUSED 127.0.0.1:5432` 或 `password authentication failed`

**检查清单：**

1. Docker 跑了吗？→ `docker ps --filter "name=syncai"` 看容器状态
2. 容器状态是 `healthy` 吗？如果不是，等 5-10 秒，PostgreSQL 启动需要时间
3. 跑了 `npm run db:migrate` 吗？
4. 如果彻底乱套了：`npm run db:down -v` 删掉数据卷，再 `npm run db:up`，最后 `npm run db:migrate`

### 3.1 切换电脑后会话创建失败 / 提示工作目录无效

**现象：** 创建会话时报“working directory 未配置”或“目录在当前机器不存在”

**原因：** `working_directory` 本质上就是当前电脑上的本地项目路径。数据库跟着迁到新电脑后，原来那台电脑的路径不一定还有效。

**解决：**

1. 确认这个项目在当前电脑上的真实本地路径
2. 把项目的 `working_directory` 更新成**新电脑上的真实路径**
3. 确保该目录存在，并且就是你希望 Codex 执行的项目目录

> 当前前端页面只支持**创建项目时填写工作目录**；如果项目已经存在而且路径失效，需要通过后续补充的编辑能力、接口或数据库修正这个字段

> 现在系统不会再偷偷退回到服务进程的当前目录执行；工作目录缺失或失效时会直接报错，避免在错误目录里跑真实 Codex。

### 4. `npm install` 报错

**现象：** `npm ERR! Unsupported engine`

**解决：** 确认 `node -v` 输出 ≥ 24。Windows 用户如果装了 nvm：`nvm install 24 && nvm use 24`。

### 5. 前端页面空白 / 控制台报 502

**现象：** 浏览器打开 5173 看到空白页，F12 看 Network 里 `/api` 请求报 502

**解决：** 后端（3001）没有启动。再开一个终端跑 `npm run dev:server`，或者确认 `npm run dev` 的输出中后端服务没有报错。

## 完整流程速查

```bash
# 1. 装依赖
npm install

# 2. 启动数据库
npm run db:up

# 3. 建表
npm run db:migrate

# 4. 启动前后端
npm run dev

# 5. 浏览器打开 http://localhost:5173
```

## 环境变量（可选）

项目根目录有 `.env.example`，如果需要自定义配置，复制为 `.env` 后修改：

```bash
cp .env.example .env
```

常用变量：

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `PORT` | `3001` | 后端端口 |
| `SYNCAI_CODEX_PATH` | 空 | 手动指定 Codex 可执行路径；留空时默认使用系统里的 `codex` |
| `SYNCAI_POSTGRES_DB` | `syncai` | 数据库名 |
| `SYNCAI_POSTGRES_USER` | `syncai` | 数据库用户 |
| `SYNCAI_POSTGRES_PASSWORD` | `syncai` | 数据库密码 |
| `SYNCAI_POSTGRES_PORT` | `5432` | 数据库端口 |
| `SYNCAI_REDIS_PORT` | `6380` | Redis 端口 |

### Codex 路径说明

- 默认直接使用当前电脑上已经安装好的 `codex`
- 如果你的安装位置特殊，可以在 `.env` 里手动配置：

```bash
SYNCAI_CODEX_PATH=C:\\tools\\codex.exe
```

### 换电脑时的最小检查清单

1. `npm install` 成功
2. `npm run doctor` 通过
3. Docker 里的 PostgreSQL / Redis 已启动
4. `codex --version` 可执行，或 `npm run doctor -- --with-codex-exec` 通过
5. 项目的 `working_directory` 已改成当前电脑上的真实目录
