# SyncAI 文档说明

**更新时间：** 2026-04-15

## 1. 当前正式文档规则

从当前版本开始，`docs/` 下的正式文档一律使用中文文件名。

适用范围：

- 产品基线文档
- 架构与设计文档
- 数据库与 API 文档
- 测试与开发拆解文档

处理原则：

- 已有英文命名旧文档先保留，不立即删除
- 后续新增正式文档不再使用英文文件名
- 如需保留英文命名文件，只能放在历史参考或归档语境中

---

## 2. 当前正式文档体系

当前应作为开发主入口的中文正式文档如下：

- `产品需求文档.md`：产品边界与 MVP 定义
- `MVP架构设计.md`：MVP 技术架构基线
- `Agent接入设计.md`：Codex 接入与事件归一化规则
- `概要设计.md`：系统对象、状态、流程、模块边界总入口
- `数据库设计.md`：正式数据库设计基线
- `API设计.md`：正式 API 与实时事件设计基线
- `开发任务拆解.md`：正式开发实施顺序与里程碑
- `测试用例设计.md`：测试用例基线
- `测试脚本设计.md`：测试脚本与测试工程结构基线

其中：

- 产品口径以 `产品需求文档.md` 为准
- 设计收口以 `概要设计.md` 为准
- 数据和接口落地以 `数据库设计.md`、`API设计.md` 为准
- 实施顺序以 `开发任务拆解.md` 为准
